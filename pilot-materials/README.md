# pilot-materials/

Operational documents needed to actually run the n=10 usability/calibration pilot specified in [`../pilot-protocol.md`](../pilot-protocol.md). These are *templates* — they need to be finalized against the academic co-PI's institutional IRB requirements and the chosen recruitment platform's conventions before use.

## Files

- [`consent-form.md`](consent-form.md) — informed-consent template. Includes the candor moment (this is a pilot for an unvalidated instrument) explicitly, per the project's trust premise. To be reviewed and adapted for IRB submission.
- [`recruitment-script.md`](recruitment-script.md) — Prolific (or equivalent panel) listing template. Includes the screening criteria, time commitment, compensation structure, and the pre-pilot expectations.

## Open

These templates are starting points, not the final filed documents. Anyone running the pilot would need to:

1. Recruit an academic co-PI at an institution with an IRB
2. Submit the consent form (likely with revisions per institutional template requirements)
3. Get IRB exemption determination or full approval as applicable
4. Localize the recruitment listing for the chosen platform's conventions
5. Pre-register the analysis plan at OSF (see [`../pre-registration.md`](../pre-registration.md))

Approximate timeline from "co-PI identified" to "recruitment open": 8-12 weeks, dominated by IRB.
